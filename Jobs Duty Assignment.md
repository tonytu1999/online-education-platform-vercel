# Jobs Duty Assignment

## 1. 角色分工总表

| 成员 | 主要职责 | 具体工作内容 | 交付物 |
|---|---|---|---|
| 成员 1 | 课程内容 / 数据编排 / PPT 制作 | 整理课程知识点与演示素材；组织示例数据；统一页面展示口径；负责最终汇报 PPT 的结构、文案和视觉呈现；协调 presentation 讲述顺序 | 课程内容包、示例数据、最终 PPT |
| 成员 2 | 移动端开发 | 负责 Flutter 移动端整体展示；实现学生端与家长端切换；完成学生学习主流程；完成家长查看汇总信息的主流程；处理基础页面联动 | 学生端 / 家长端移动应用原型 |
| 成员 3 | 教师端 / 学校端网页开发 | 负责 React 网页端；实现教师与学校管理员视角；完成班级与学校数据展示；实现基础后台看板与页面结构 | 教师端 / 学校端网页应用原型 |
| 成员 4 | 后端开发 | 负责 Node.js / Express / MySQL 后端；实现登录与角色权限；提供前端所需 API；串联学习进度、用户关系与 AI 调用；保证各端数据一致 | 后端服务、数据库结构、API 接口 |

## 2. 各成员职责细化

| 成员 | 细化职责 |
|---|---|
| 成员 1 | 负责把 PRD 中的课程主题、知识点层级和展示逻辑整理成适合演示的内容；准备示例数据，保证学生端、家长端、教师端展示的数据口径统一；将最终项目包装成一个适合课堂汇报的故事线；输出最终 PPT，并统筹 presentation 节奏。 |
| 成员 2 | 负责移动端的整体用户体验展示，重点是学生端和家长端之间的身份切换；学生端侧重学习路径、AI 对话、能力测试和进度展示；家长端侧重学习汇总与查看入口；不追求完整产品级细节，重点是主流程可演示、页面可串联。 |
| 成员 3 | 负责教师端和学校端的网页页面呈现，重点做出管理视角和数据看板；展示班级列表、学生概况、掌握率、学习进度等核心信息；如果时间有限，优先保证“看起来完整、逻辑清楚、数据能展示”，而不是做复杂交互。 |
| 成员 4 | 负责后端核心能力，确保前端页面都能拿到统一的数据；包括用户登录、角色区分、基础关系数据、学习进度数据、AI 接口调用与返回、以及课程内容相关接口；如果资源有限，优先保证演示链路能跑通，后端实现可以偏轻量。 |

## 3. 项目 Milestones

| 时间 | Milestone | 说明 |
|---|---|---|
| 5月17日 | 完成主要功能核心开发 | 完成学生端、家长端、教师/学校端、后端的主要核心功能，保证主流程可展示。 |
| 5月24日 | 完成 API 接口、PPT 与主要演示流程 | 完成所有关键 API 接口联通，同时把 PPT、讲述顺序和主要演示链路准备好。 |
| 5月29日 | 最终 Presentation | 完成最终汇报展示，确保页面、数据、讲稿和演示流程都可顺利呈现。 |

## 4. 补充说明

这个分工适合 MVP 和课程展示场景，重点是“主流程可展示、叙事清楚、页面完整”。如果后续还有时间，可以再补测试、优化和功能扩展，但当前阶段先把演示闭环做出来最重要。
