# 移动端界面设计与功能需求（MVP）

## 1. 文档依据
- `CLAUDE.md`：遵循MVP优先、演示优先、简单可串联原则。
- `Jobs Duty Assignment.md`：移动端聚焦学生端与家长端主流程、身份切换与页面联动。
- `PRD.md`：以学生端/家长端用户故事与验收标准为功能基线。

## 2. 设计目标与范围
- 目标：在 Flutter 移动端完成“学生学习闭环 + 家长查看汇总”的可演示原型。
- 范围：学生端、家长端、登录认证、角色切换、学习主流程、家长监控主流程、心理健康占位页。
- 非范围（MVP不做或仅占位）：
  - 推送通知能力（仅预留入口，不做真实接入）
  - 心理健康真实分析（仅占位UI与空数据态）
  - 语音输出（仅文字回复，语音输入可做）

## 3. 移动端信息架构

### 3.1 全局结构
- 启动页 / 欢迎页
- 登录注册页（手机号/邮箱）
- 角色选择页（学生 / 家长）
- 学生端主框架（底部导航）
- 家长端主框架（底部导航）
- 设置页（账号、退出、订阅占位）

### 3.2 学生端导航（Bottom Navigation）
1. 学习：学科→章节→知识点
2. AI助手：自由对话 / 知识点引导对话
3. 能力测试：考试分数、错题、测试入口
4. 进度：掌握度与学习汇总
5. 我的：个人资料、组织关联、设置

### 3.3 家长端导航（Bottom Navigation）
1. 总览：孩子学习汇总
2. 学科进度：分学科掌握情况
3. 心理健康：占位页（聚合指标空态）
4. 孩子管理：绑定/切换孩子
5. 我的：账号设置、订阅占位

## 4. 界面设计要求

### 4.1 统一设计规范
- 风格：简洁、教育场景、数据可读性优先。
- 颜色语义：
  - 已掌握：绿色
  - 部分掌握：橙色
  - 未掌握：灰/红（按视觉规范统一）
  - 心理风险：低/中/高使用绿/黄/红
- 通用组件：
  - 进度卡片、学科卡片、知识点列表项、趋势图容器、空状态组件、权限提示组件。

### 4.2 关键页面（学生端）
1. 登录/注册页  
   - 手机号+验证码、邮箱+密码。  
   - 登录成功后进入加入学校/组织引导页（与PRD保持一致）。

2. 学习首页（学科页）  
   - 展示语文/数学/英语。  
   - 点击进入章节列表，再进入知识点列表。  
   - 知识点项展示：名称、描述、当前掌握标签。

3. AI助手页  
   - 顶部模式切换：自由对话 / 知识点引导。  
   - 聊天区：消息流、输入框、语音输入按钮。  
   - 需展示禁用话题拦截提示（安全学习）。

4. 能力测试页  
   - 入口分区：录入考试分数、录入/拍照错题、开始测试。  
   - 结果页：分数、知识点掌握变化、趋势图（时间轴）。

5. 学习进度页  
   - 按学科→章节→知识点层级展开。  
   - 展示掌握等级、学习时长、完成率。

### 4.3 关键页面（家长端）
1. 登录/注册页  
   - 同学生端登录方式。  
   - 登录后进入孩子绑定流程（手机号/邮箱/二维码）。

2. 孩子总览页  
   - 汇总指标：完成率、知识点掌握率、学习时长。  
   - 最近7天趋势图。  
   - 明确提示“不可查看孩子AI对话内容”。

3. 分学科进度页  
   - 语文/数学/英语分卡片展示掌握率。  
   - 可进入科目详情查看章节级汇总（仅聚合数据）。

4. 心理健康页（占位）  
   - 指标位：情绪极性、压力关键词、风险等级。  
   - 默认空数据态与“后续接入AI分析”提示。

5. 孩子管理页  
   - 支持多孩子绑定与切换。  
   - 展示绑定状态、最近同步时间。

## 5. 功能需求（按优先级）

### 5.1 P0（必须完成）
- 角色登录（手机号/邮箱）
- 学生学习路径：学科→章节→知识点
- AI交互统一入口（自由对话 + 知识点引导）
- 学生能力测试主流程（分数/错题/测试结果展示）
- 学生学习进度层级展示（掌握度标签）
- 家长绑定孩子
- 家长学习汇总看板（完成率、掌握率、学习时长、7天趋势）
- 权限隔离：家长与教师均不可查看学生原始AI对话

### 5.2 P1（应完成/可占位）
- 第三方登录入口占位（微信/QQ）
- 学生语音输入（语音转文字）
- 心理健康页面UI与空状态（家长端）
- 订阅/支付前端页面占位（按钮“即将上线”）

### 5.3 暂缓项（MVP后）
- 推送通知真实接入（PRD澄清为MVP不实现）
- 语音输出
- 心理健康AI分析与高风险告警联动

## 6. 关键业务规则
- 掌握度分级：未掌握 / 部分掌握 / 已掌握。
- 掌握度更新依据：测试结果 + AI对话评估（按PRD定义）。
- 敏感话题过滤：色情、暴力、犯罪等黑名单，支持学科上下文规则。
- 家长查看边界：仅聚合数据，不可查看原始对话。
- 心理健康数据边界：仅展示聚合结果，不展示原始情绪数据。

## 7. 页面联动与演示主流程

### 7.1 学生端演示链路
登录 → 选择学科/知识点 → AI引导学习 → 能力测试 → 进度更新展示。

### 7.2 家长端演示链路
登录 → 绑定孩子 → 查看学习总览/趋势 → 查看分学科进度 → 查看心理健康占位页。

### 7.3 角色切换链路
“我的”页面提供角色切换入口，切换后加载对应导航与数据视图。

## 8. 数据展示与空状态要求
- 所有图表支持空态：显示“暂无数据”与引导文案。
- 心理健康模块默认空态，不报错、不崩溃。
- 接口失败时展示轻提示并允许重试，不阻断主流程演示。

## 9. 验收口径（移动端）
- 主流程可完整演示，页面可串联、跳转正确。
- 学生端与家长端核心页面齐备，角色边界清晰。
- 文案、结构与PRD用户故事一致，不扩展超出MVP范围。
- 满足课堂展示目标：叙事清楚、页面完整、数据口径统一。
