**版本**：v1.0 MVP  
**日期**：2026年5月8日  
**产品经理**：资深产品经理（Series A初创公司）  
**目标上线日期**：2026年5月20日（MVP）

## 一、产品概述

### 1.1 产品愿景
打造AI-Agent驱动的个性化在线教育平台，通过苏格拉底式提问引导学生思考，提供24小时个性化学习指导，同时监测学生心理健康状态，让家长、教师、学校实时掌握学生学习进度与心理状态。

### 1.2 核心痛点
- **个性化缺失**：传统教育平台无法提供完全个性化的教育指引
- **课后指导空白**：学生课后自学时无法24小时获得老师指导
- **心理健康盲区**：学生心理问题难以及早发现和干预
- **家校沟通断层**：家长、教师对学生学习进度和心理健康缺乏实时监测手段

### 1.3 解决方案
- **AI学习助手**：基于大模型API（Claude/OpenAI/DeepSeek等），通过苏格拉底式提问引导学生独立思考
- **学习进度追踪**：按学科→章节→知识点层级展示掌握程度（AI自动判断）
- **多端监控面板**：家长端（移动端）查看汇总数据，教师/学校端（网页端）查看班级/全校数据
- **心理健康监控**：多端心理健康监控面板——家长端查看孩子心理状态，教师端监控班级学生心理健康，学校管理员查看全校/全年级数据；前端页面先行，后续接入AI对话情绪分析（情绪极性、压力关键词、风险等级）

### 1.4 目标用户
| 用户角色 | 使用端 | 核心需求 |
|---------|--------|---------|
| 学生 | 移动端 | 个性化学习、课后自学、能力测试 |
| 家长 | 移动端 | 监控孩子学习进度、心理健康状态（汇总数据） |
| 教师 | 网页端 | 查看班级学生知识点掌握率、平均学习进度 |
| 学校管理员 | 网页端 | 查看全校/全年级数据（权限高于教师） |
| 教育局 | 无 | 仅用于项目支持，不做功能开发 |

## 三、功能需求与验收标准（Acceptance Criteria）

### 3.1 学生端 - 移动端（React Native）

#### 功能模块：用户认证
**描述**：学生通过手机号+验证码/邮箱+密码/第三方登录（微信、QQ）注册登录  
**验收标准**：
- [ ] 支持手机号+验证码登录（短信服务集成）
- [ ] 支持邮箱+密码登录
- [ ] 支持微信、QQ第三方登录
- [ ] 登录后引导用户进入"加入学校/组织"页面
- [ ] Token机制保持登录状态

#### 功能模块：学习主题选择（学科→章节→知识点）
**描述**：学生按层级选择学习内容，内置课程库包含语文、数学、英语  
**验收标准**：
- [ ] 首页展示学科列表（语文、数学、英语）
- [ ] 点击学科展开章节列表
- [ ] 点击章节展开知识点列表（带知识点描述文本）
- [ ] 每个知识点关联练习题（文本格式）
- [ ] 选择知识点后进入AI对话界面

#### 功能模块：AI交互模块（自由对话 + 苏格拉底式引导）
**描述**：学生通过统一的AI交互入口，既可进行自由提问对话，也可在知识点学习时获得苏格拉底式提问引导，两种模式无缝切换  
**验收标准**：
- [ ] 提供统一的AI交互入口（底部导航栏），支持自由对话模式和知识点引导模式
- [ ] 自由对话模式：学生可自由输入问题，不限定知识点范围
- [ ] 知识点引导模式：学生选择知识点后，AI通过苏格拉底式提问引导学生思考（不直接给答案）
- [ ] 支持文字输入+语音输入（语音转文字）
- [ ] AI回复支持文字展示（语音输出MVP预留）
- [ ] 预设禁用话题过滤（色情、暴力、犯罪等关键词黑名单，分学科上下文感知）
- [ ] 对话历史记录保存（自由对话与引导对话分别记录）
- [ ] 集成多个AI模型API（Claude、OpenAI、OpenRouter、DeepSeek）
- [ ] 后台可配置当前使用的模型（支持按成本+学科适配切换）
- [ ] AI根据对话判断学生对知识点的掌握程度（标记：未掌握/部分掌握/已掌握）
- [ ] 对话上下文保持（多轮对话）

#### 功能模块：能力测试（结合考试分数与错题）
**描述**：结合学生在校考试分数及错题数据，通过App内测试动态评估并展示知识点掌握程度的浮动变化  
**验收标准**：
- [ ] 提供"能力测试"入口
- [ ] 支持学生导入/录入在校考试分数（按学科、考试类型）
- [ ] 支持学生录入或拍照识别错题（按知识点归类）
- [ ] 测试题目从知识点关联练习题中抽取，并根据错题记录智能加权出题
- [ ] 测试完成后AI综合考试分数+错题数据+测试结果自动评估当前水平
- [ ] 根据考试结果动态展示知识点掌握程度的浮动变化（如：某知识点因考试失分而下降，因测试正确而回升）
- [ ] 以可视化图表展示掌握程度变化趋势（时间轴+浮动曲线）
- [ ] 测试结果同步到学习进度展示

#### 功能模块：学习进度展示
**描述**：按学科→章节→知识点层级展示掌握程度  
**验收标准**：
- [ ] 学习进度页面按学科展开折叠
- [ ] 每个学科下展示章节，章节下展示知识点
- [ ] 每个知识点显示掌握程度（进度条或标签：未掌握/部分掌握/已掌握）
- [ ] 掌握程度由AI自动判断并更新
- [ ] 展示学习时长、完成率等汇总数据

## 四、技术栈推荐（优化速度）

## 八、已确认实施细节（2026-05-09 澄清后）

### 8.1 AI逻辑规则
- 苏格拉底式提问：采用固定全局系统提示词强制风格
- 知识点掌握度评估：测试分数 + AI判断对话成功率（对话+测试结合）
- 模型切换逻辑：默认按成本+学科适配切换，无特殊偏好
- AI接口降级：接口不可用/超时仅返回错误提示，无备用模型自动切换
- 敏感词过滤：采用基于规则的分学科上下文感知过滤（如生物课允许"性"相关术语）

### 8.2 内容范围
- 覆盖学段：小学（1-6年级）、初中（7-9年级）、高中（10-12年级）
- 课程体系：香港DSE + 中国义务教育课程标准
- 免费版限制：免费用户每日最多访问5个知识点
- 课程资源：需寻找对应课程标准的开源知识点、练习题资源

### 8.3 隐私与权限
- 合规要求：需获取家长知情同意、用户注册时自主申报年龄、账号注销后数据保留3年（默认）
- 家长-孩子绑定：家长验证通过后自动绑定，无需孩子确认
- 角色权限：
  - 学校管理员：可手动编辑学生知识点掌握等级
  - 教师：无法查看学生与AI的对话内容
  - 家长：无法查看任何对话内容，仅可查看聚合学习数据
- 心理健康模块：不存储/展示原始情绪数据，仅展示聚合风险报告

### 8.4 技术实施要求
- 开发流程：先定义核心数据库表结构、API接口规范，再启动编码
- 移动端兼容：React Native支持Android 8+、iOS 13+，无离线使用支持
- 推送通知：MVP不实现任何推送功能
- 第三方登录：MVP可选微信登录，默认仅支持手机号、邮箱登录
- 技术栈：按原PRD第四章节选型，无调整

### 8.5 边界情况处理
- 范围裁剪：无特殊优先级，非核心功能可按需裁剪
- 敏感词误判：采用分学科规则过滤，避免上下文误判（如生物课术语）
- 模型切换：无用户偏好，默认采用成本+学科适配逻辑
